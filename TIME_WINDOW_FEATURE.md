# 性能抖动图 · 时间窗格滑动查看功能

## 📍 入口

任务列表 → 任意任务 → "性能数据" 按钮 → 打开 `TaskMetricsDialog` 对话框。

对话框中的四张抖动图（IOPS / 带宽 / 延迟 / CPU）现已支持按时间窗格查看历史抖动。

## 🎛️ 三种交互方式

| 交互 | 操作 | 效果 |
|------|------|------|
| **快捷按钮** | 点击顶部"最近 1/5/15 分钟"或"最近 1 小时"或"全部" | 四张图同步跳到该时间窗口 |
| **底部时间滑块** | 拖动任一图表底部的时间滑块两端或中间 | 四张图同步滑动到对应时间 |
| **图表内滚轮/拖拽** | 在任一图表上用鼠标滚轮缩放，或按住拖拽 | 四张图同步缩放/平移 |

## 🔗 四张图联动

通过 `echarts.connect(group)` 把四张图放进同一个 group，任意一张图的 dataZoom 动作会自动同步到其他三张。这样你可以在 IOPS 图上看到某个尖刺时，一眼扫到同时段的带宽、延迟、CPU 是否也出现了异常。

## 🟢 实时跟随 vs 🔒 手动锁定

|  | 实时跟随模式 | 手动锁定模式 |
|---|---|---|
| **进入条件** | 默认 / 点击"全部" / 点击"回到实时" | 用户手动拖动任一图表 / 点击非"全部"的范围按钮 |
| **窗口行为** | 每次有新数据，窗口自动跟到最新时间，宽度保持不变 | 窗口固定在用户选定的时间段，新数据到来不会跳走 |
| **UI 标识** | 右上角显示绿色 "跟随最新数据" | 右上角显示黄色 "已锁定时间窗格 · 不跟随实时"，并出现 "回到实时" 按钮 |

这个设计解决了"实时刷新 vs 查看历史"的冲突：用户可以在实时监控中随时拖回去看历史抖动，而实时数据仍在后台继续采集写入历史缓存，不会丢。

## 🔧 抖动分析专项优化

普通趋势图和抖动图的区别在于"细节保留"，这版做了四点专门优化：

1. **关闭 `smooth`**：平滑曲线会把真实的毛刺磨平，抖动分析场景必须关掉
2. **启用 `sampling: 'lttb'`**：数据量大时使用 LTTB 下采样算法，保留视觉上的尖刺/谷底，不会抹平异常点
3. **关闭 `animation`**：实时数据频繁更新时动画会让画面晃动，干扰视觉
4. **扩容历史缓存到 600 点**（约 20 分钟 @ 2s 采样），让时间窗格有足够多的数据可滑

## 🧮 时间轴类型

从 `type: 'category'` 改为 `type: 'time'`，数据格式从 `[v1, v2, ...]` 改为 `[[ts1, v1], [ts2, v2], ...]`。

好处：
- 采样间隔不均匀时（比如网络抖动导致某次采样延后），时间轴按真实时间展示，不会出现"本应 10:00:05 的数据贴在 10:00:02 旁边"的错位
- dataZoom 的快捷范围按钮可以直接按"最近 N 秒"计算 `startValue/endValue`，不用反推百分比
- 跨采样周期的抖动周期性分析更准确

## 📦 接口依赖

前端调用：
```
GET /api/tasks/{task_id}/metrics?limit=600
```
需要返回按时间顺序排列的性能数据数组：
```json
[
  {"timestamp": 1734567890000, "iops": 15234, "bandwidth": 520.5, "latency": 1.23, "cpu_usage": 35.6},
  ...
]
```
字段：
- `timestamp` — 毫秒时间戳（数字），或任何 `Date.parse` 能解析的字符串
- `iops` — 数字，IOPS
- `bandwidth` — 数字，**单位 MB/s**（不是字节！参见 `task_service.py` 里 `total_bw_mbs` 的计算）
- `latency` — 数字，微秒
- `cpu_usage` — 数字，百分比（0-100）

后端若尚未实现该接口，可参考 `BUG_FIX_REPORT.md` 末尾的代码片段在 `api/tasks.py` 中补上。

## 🎨 带宽单位修复

本次同步修复了带宽显示的单位 bug：后端存的是 MB/s，但前端一直在用 `formatBytes`（按字节格式化），会把 500 MB/s 显示成 "500 B/s"。

新增了 `formatMBps` 函数专门处理带宽：
- `>= 1024 MB/s` 显示为 GB/s
- `>= 1 MB/s` 显示为 MB/s
- `>= 1 KB/s` 显示为 KB/s
- 否则显示为 B/s

所有带宽卡片、统计区、图表 Y 轴和 tooltip 都已替换成 `formatMBps`。

## 🧪 使用场景示例

**场景 1：定位压测中的一次性能突降**

1. 实时监控中看到 IOPS 突然下降
2. 鼠标滚轮放大那个时间段，或点击"最近 1 分钟"
3. 同步查看四张图：延迟是否飙升？CPU 是否打满？带宽是否同步下降？
4. 通过底部时间滑块继续滑动前后几分钟，判断是一次性尖刺还是持续性问题

**场景 2：对比压测前后的性能基线**

1. 拖动底部滑块把时间窗格定在压测开始前的 5 分钟
2. 切换 "最近 5 分钟" 快捷按钮到"回到实时"，查看当前压测中的数据
3. 肉眼对比两段时间的抖动幅度和均值差异

**场景 3：查看长时间任务的周期性抖动**

1. 点击"最近 1 小时"
2. 观察 IOPS/延迟是否呈规律的周期性抖动（可能是 GC、日志轮转等）
3. 聚焦到某个周期放大查看，四张图联动观察根因
