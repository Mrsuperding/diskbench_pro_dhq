# DiskBench Pro 测试计划

## 一、测试范围

### 1.1 测试模块
| 模块 | 功能 | 优先级 |
|------|------|--------|
| 认证模块 | 登录/注册/登出/令牌刷新 | P0 |
| 节点管理 | 节点的 CRUD/分区管理/健康检查 | P0 |
| 用例管理 | FIO 用例的 CRUD/模板管理 | P0 |
| 任务管理 | 测试任务创建/启动/停止/监控 | P0 |
| 监控模块 | 实时 IO 监控/历史数据查询 | P1 |
| 调度任务 | 定时任务创建/修改/删除 | P1 |
| 性能基准 | 基准配置/对比分析 | P1 |
| 告警管理 | 告警规则/告警事件 | P2 |
| 审计日志 | 操作日志查询 | P2 |
| 用户管理 | 用户 CRUD/角色管理 | P1 |

### 1.2 测试类型
- **API 测试**：后端 API 接口功能验证
- **UI 测试**：前端页面交互和用户体验

## 四、执行计划

### 4.1 优先级排序
1. **P0**：核心流程，必须通过（认证、节点列表、用例列表、任务创建/启动/停止）
2. **P1**：重要功能，应通过（节点 CRUD、用例 CRUD、调度、用户管理）
3. **P2**：辅助功能，尽量通过（告警、审计、克隆）

### 4.2 测试环境要求
- 后端服务运行在 `http://localhost:8000`
- 前端服务运行在 `http://localhost:3000`
- MySQL 数据库可访问
- Redis 服务可访问

### 4.3 测试数据要求
- 测试用户：`admin/admin123`（管理员）、`demo/demo123`（普通用户）
- 测试数据使用独立环境，避免污染生产数据
